# Naming Constraints

- Use Nest conventions: `*.module.ts`, `*.controller.ts`, `*.service.ts`, `*.spec.ts`.
- Name folders and route segments in lowercase kebab case; class names in PascalCase; functions/values in camelCase.
- Use domain names, not technology names, for new folders (for example `users/`, not `controllers/`).
- Make test descriptions state behavior, following the existing `AppController` suite style.
